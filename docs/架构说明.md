# 架构说明

流水线分为四层：

1. `inventory` 递归清点文件，并调用格式解析器。
2. `analyze` 从素材证据和项目配置生成 Brand Lock、对标、缺图和图片规划。
3. `pipeline` 组织阶段并形成唯一结构化结果。
4. `report` 从同一结果渲染 6 份 Markdown 和 1 份 JSON，防止跨文档不一致。

配置优先级为“人工配置 > 可解析素材证据 > 行业默认建议”。联网结果只作为对标候选，不会自动写入 Brand Lock。

解析器不把 ZIP 内容释放到磁盘，降低路径穿越风险。单个文件解析失败会保留在素材清单并记录警告，不中断其余项目分析。
