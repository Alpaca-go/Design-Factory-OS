# 使用手册

## 推荐工作流

1. 执行 `init` 创建配置和 `assets/` 目录。
2. 放入原始素材并完善 `design-factory.json` 中已确认的品牌事实。
3. 先执行 `inventory` 检查文件是否可读。
4. 执行 `analyze`；需要实时案例候选时添加 `--online`。
5. 人工复核 Brand Lock 中的待确认项，并更新配置后重新生成。
6. 按图片规划制作，最后把 Chat 生图任务包交给执行者。

## 支持范围

| 格式 | 能力 |
|---|---|
| ZIP | 读取中央目录、文件名与数量，不解压到磁盘 |
| PDF | 页数、基础元数据与可见文本线索；扫描 PDF 仅能统计页数 |
| PPTX/POTX/PPSX | 页数、幻灯片文本、主题色与主题字体 |
| 旧版 PPT | 纳入清单，暂不深度解析 |
| PNG/JPEG/GIF/WebP/SVG/BMP | 格式和可用尺寸；SVG 额外提取颜色与文本 |
| MD/TXT/JSON/CSV | 文本线索，最多读取前 200KB |

## 常见问题

### 为什么没有自动确定主色？

主色是品牌事实。只有配置、PPTX 主题或 SVG 明确色值时，系统才输出候选；仅凭压缩图片文件名无法可靠判定时会要求人工确认。

### 联网失败是否阻塞？

不阻塞。系统记录失败原因并回退到内置、按行业分类的策展案例库。生产项目应人工复核联网候选的质量与可访问性。

### 可重复运行吗？

可以。默认输出目录在扫描时会被忽略，避免上一次报告被当作新素材。相同输入和配置在离线模式下产生一致的业务数据，仅生成时间不同。
